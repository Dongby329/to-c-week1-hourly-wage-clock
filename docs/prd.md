## 1. 产品概述

### 1.1 产品一句话

**让打工人的每一秒都看得见价值——温和、数据化、有仪式感的时薪实时时钟。**

### 1.2 产品定位

时薪桌面钟是一款**纯 Web 端的桌面常驻时薪实时追踪工具**。用户只需设置一次月薪和工作时间，系统自动检测当前日期和时间状态，智能切换"工作前倒计时 → 工作中实时累加 → 工作后定格日薪"三种模式，零操作即可获得沉浸式的"时间就是金钱"体验。

### 1.3 核心差异化

| 维度 | 竞品 | 我们 |
|------|------|------|
| 时间感知 | 手动启动/停止 | **智能三态自动检测** |
| 安装门槛 | Chrome 扩展 / App | **纯 Web，零安装** |
| 调性 | 冷数据 / 炫技 | **温和激励 + 数据美学** |
| 节假日 | 无 | **内置法定节假日 + 调休** |
| 定制化 | 固定 | **语言/币种/主题可切换** |

### 1.4 目标平台

- **主平台**：桌面 Web（Windows / macOS / Linux 浏览器）
- **扩展**：PWA 支持（后续迭代）
- **明确不做**：原生 App、Chrome 扩展（现阶段）

## 3. 核心用户旅程

### 3.1 首次使用旅程

```
打开页面 → 登录/注册（输入用户名）
  → 设置向导 Step 1：输入月薪 + 选择币种
  → 设置向导 Step 2：选择工作制度（双休/大小周/单休） + 勾选工作日
  → 设置向导 Step 3：设定工作时间段 + 加班费率
  → 保存 → 自动跳转时钟主页
  → 系统自动判断当前状态（上班前/中/后）→ 开始追踪
```

### 3.2 日常使用旅程（已有设置）

```
打开页面 → 登录 → 直接进入时钟主页
  → 系统自动检测：日期（工作日/节假日）+ 时间（前/中/后）
  → 工作中：看到薪资每秒跳动，可选暂停/加班
  → 下班后：日薪定格，查看日历回顾本月累计
```

### 3.3 返回用户 - 设置修改旅程

```
进入设置页 → 看到当前设置总览（只读卡片）
  → 点击"Edit Settings" → 向导预填已有值
  → 修改 → 保存 → 返回时钟
```

### 4.2 登录系统

#### 4.2.1 账号机制

- **本地账号**：基于 `localStorage` 的纯前端账号系统，无需后端
- **用户标识**：用户名作为唯一标识，存储键格式 `wageClock_<username>_<key>`
- **会话管理**：`sessionStorage` 存储当前登录用户，关闭标签页即登出
- **多账号切换**：设置页底部提供 "⇄ Switch Account" 按钮

#### 4.2.2 登录页面

- 输入用户名
- 新用户自动注册（首次使用） / 已有用户恢复数据
- 已注册用户以 chip 形式展示，点击快速登录
- 不支持密码（本地工具，非安全场景）

### 4.3 时钟主页（核心页面）

#### 4.3.1 实时薪资显示

```
┌──────────────────────────────────┐
│          ¥ 681.82                │  ← hero-sized 金额 (clamp 72px-136px)
│        earning · live            │  ← 状态标签
│      08:32:15 today              │  ← 今日已工作时长
│       ¥0.0237 / sec              │  ← 每秒速率
└──────────────────────────────────┘
```

- **字体**：大金额使用系统字体，`font-weight: 700`，`letter-spacing: -4px`
- **动画**：每秒 `requestAnimationFrame` 驱动累加
- **精度**：显示到小数点后 2 位，内部计算精度到 4 位

#### 4.3.2 状态指示器 (Pill)

顶部 pill 组件实时显示当前状态：

| 状态 | Pill 文字 | 颜色 | Dot |
|------|-----------|------|-----|

| 工作中 (working) | `earning` | 绿色 `#34d399` | 绿色 |
| 已暂停 (paused) | `paused` | 蓝色 `#60a5fa` | 蓝色 |
| 已下班 (after) | `done` | 灰色 | 灰色 |
| 加班中 (overtime) | `overtime` | 红色 `#f87171` | 红色 |
| 休息日 (rest) | `rest day` | 灰色 | 灰色 |

#### 4.3.3 操作按钮

| 按钮 | 显示条件 | 行为 |
|------|----------|------|
| ⏸ Pause | 工作中 | 冻结累加，保持当前金额 |
| ▶ Resume | 暂停中 | 从冻结金额继续累加（不补偿暂停时间） |
| Start Overtime | 已下班 | 开始按加班费率累加 |
| Stop Overtime | 加班中 | 定格加班收入，可重新开始 |

- **暂停按钮**：大尺寸（`padding: 14px 56px, font-size: 20px, min-width: 180px`），蓝色边框
- **位置**：金额下方，`margin-top: 48px`

#### 4.3.4 加班模式

- **触发**：工作后状态下手动点击 "Start Overtime"
- **费率**：从设置中读取，支持三种模式：
  - 倍率模式（1.0x / 1.5x / 2.0x / 3.0x）
  - 基础速率模式（使用正常每秒薪资）
  - 自定义模式（¥/小时 或 ¥/秒 或 ¥/次）
- **显示**：与正常工作时间累加，金额计入当天总收入

### 4.7 自定义光标 (TargetCursor)

#### 4.7.1 组件规格

- **来源**：React Bits `<TargetCursor />`，完整 vanilla JS 移植
- **结构**：中心 4px 圆点 + 4 个 L 形角括号（12px × 12px，3px 边框）
- **空闲动画**：持续旋转（可配置，默认 2s/圈）
- **目标吸附**：hover `.cursor-target` 元素时，四角吸附到元素四角
- **视差效果**：吸附后角括号带延迟跟随（`parallaxOn: true`）
- **颜色过渡**：可配置目标颜色（`cursorColorOnTarget`）
- **点击反馈**：dot 缩至 0.7x，整体缩至 0.9x
- **混合模式**：`mix-blend-mode: difference`，自动反色
- **移动端禁用**：自动检测触摸设备 + 小屏，不渲染
- **Containing block 补偿**：自动检测并补偿 CSS transform/filter 等导致的偏移

#### 4.7.2 标记元素

各页面已标记 `.cursor-target` 的元素：

| 页面 | 元素 |
|------|------|
| clock.html | Pause / Resume / Start OT / Stop OT 按钮 |
| calendar.html | 月份导航按钮、自定义收入输入框、+Add 按钮 |
| setup.html | Edit Settings、Save & Start、Switch Account 按钮 |

### 4.8 视觉特效

#### 4.8.1 DotGrid 背景

- **技术**：Canvas 2D，全屏固定背景
- **样式**：3px 白色圆点，28px 间距，基础透明度 0.1
- **鼠标交互**：
  - 120px 范围内圆点变亮（最高 0.55 透明度）
  - 移动时产生惯性拖尾效果
  - 点击产生冲击波，200px 范围内圆点向外弹射
- **响应式**：窗口 resize 时重建网格
- **性能**：`requestAnimationFrame` 驱动，每帧更新物理 + 重绘

#### 4.8.2 3D 倾斜卡片

- 所有页面主体卡片 (`<main>`) 应用 CSS `perspective: 800px`
- `mousemove` 时根据鼠标位置旋转（最大 ±14deg）
- `mouseleave` 时回正
- 结合 `transform-style: preserve-3d`

| PillNav 入场 | opacity + y + scale | duration: 0.5s, ease: power3 |

## 5. 状态机与业务逻辑

### 5.1 状态转换图

```
未设置 → 设置中 → 已设置 → 日历检测
                              ↓
                    ┌─ 休息日 (holiday/weekend)
                    │  显示: ¥0.00 + 休息原因
                    │
                    ├─ 工作前 (before)
                    │  显示: ¥0.00 + 倒计时
                    │  ↓ 时间到
                    │
                    ├─ 工作中 (working)
                    │  显示: 每秒累加
                    │  ├→ 暂停 (paused): 冻结累加
                    │  │    └→ 工作中: 从冻结值恢复
                    │  └→ 时间到
                    │
                    ├─ 工作后 (after)
                    │  显示: 今日日薪定格
                    │  └→ 加班 (overtime): 按加班费率累加
                    │       └→ 工作后: 停止加班，定格
                    │
                    └─ 第二天→日历检测 (重新判断)
```

### 5.2 法定节假日逻辑

- 内置硬编码 JSON 数据库，包含：
  - 法定节假日日期列表（`holidays`）
  - 调休上班日列表（`makeupWorkdays`），周末补班日视为工作日
- 优先级：调休上班日 > 法定节假日 > 周末判断
- 数据年份不匹配时降级为纯周末判断

### 5.3 每秒累加算法

```
perSecondRate = (monthlySalary / workingDaysPerMonth) / workHoursPerDay / 3600

工作开始时：
  base = 从工作开始时间到现在的已流逝秒数 × perSecondRate

animWork() 循环（requestAnimationFrame）：
  now = Date.now()
  elapsed = (now - lastTick) / 1000
  acc = base + elapsed × perSecondRate
  显示 acc.toFixed(2)
  lastTick = now
```

- **暂停**：取消 `requestAnimationFrame`，保留当前 `base` 值
- **恢复**：重置 `lastTick = Date.now()`，从 `base` 继续累加
- **回溯补偿**：页面恢复时根据已流逝时间重新计算 base

### 5.4 跨天重置

- 日期变更时（`new Date().toDateString()` 变化）自动重新日历+时间检测
- 昨日 earnings 已通过每 30 秒自动存储持久化

```json
    "customAmount": 50,
```

```
键: wageClock_alice_earnings_2026-06-25
```

| 大金额 | 系统字体，`font-weight: 700` |

#### 7.1.3 间距与圆角

| 元素 | 值 |
|------|-----|
| 卡片圆角 | `16px` |
| 按钮圆角 | `4px` - `28px`（大按钮用圆角胶囊） |
| Pill 圆角 | `9999px`（全圆） |
| 卡片 padding | `2rem 1.5rem` |
| 导航 top | `16px` |

#### 7.1.4 响应式

- 时钟金额使用 `clamp(72px, 13vw, 136px)` 自适应
- 卡片宽度使用 `clamp(320px, 82vw, 620px)`
- 移动端：暂未适配（TargetCursor 自动禁用）

### 7.2 交互规范

### 7.3 空状态与异常

|------|------|
| 未登录 | 重定向到 `login.html` |
| 无设置 | 显示设置向导 |

## 9. 非功能性需求

### 9.1 性能

| 指标 | 目标 |
|------|------|
| 首次内容绘制 (FCP) | < 1s |
| 动画帧率 | 60fps（requestAnimationFrame） |
| Canvas 重绘 | 60fps，每帧遍历全部网点 |
| 存储写入 | 节流，每 30s 最多写一次 |

### 9.2 隐私与安全

- **纯本地存储**：所有数据仅存于浏览器 localStorage
- **无网络请求**：除 CDN 加载 GSAP 外，不上传任何数据
- **无追踪**：无埋点、无分析、无广告
- **账号隔离**：不同用户名数据完全隔离（不同 key prefix）

### 9.3 可维护性

- **零构建**：无需 webpack/vite/npm，直接编辑 HTML
- **单文件组件**：nav.js、target-cursor.js 自包含，带公共 API
- **命名约定**：`uk()` 函数统一管理存储键前缀
- **降级策略**：GSAP 不可用时静态显示
