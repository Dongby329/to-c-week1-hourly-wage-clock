## 👥 二、用户趋势

### 趋势1：职场人择业观历史性转变——"平衡"首超"薪酬"

智联招聘2025年调研（3,525份有效样本）：

```
择业最在意什么？
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
工作生活平衡    ████████████████████ 64.1% ← 史上首次第一
薪酬福利        ████████████████     52.1% ← 降至第二
成长空间        █████████████        42.7%
被尊重的氛围    █████████████        41.7%
```

> **启示**：用户不想被"剥削感"笼罩，但也需要感知到"我的时间有价值"。时薪桌面钟恰好满足这个心理：它不是催你加班的工具，而是**帮你看见时间价值的伙伴**。

### 趋势2：Z世代极度在意薪酬公平

- 不迷信大厂光环，但**对薪资公平极度敏感**
- "画饼"失效——要看得见摸得着的回报
- 裸辞意愿高（34.8%），Gap接受度上升（21.8%愿休3个月+）
- "爱你老己"思潮——身体/情绪/意义感，缺一不可

> **启示**：产品调性必须是"赋能"而非"压榨"。不要做成"你又浪费了X元"，而是"你的时间值得被看见"。

### 趋势3："打工人"身份认同持续强化

- "牛马""打工人"等自嘲标签深入人心，但背后是**对劳动价值的重新审视**
- 竞品"牛马计时器"的名字本身就验证了这个文化情绪
- 用户想要的是：**知道自己每一秒值多少钱**——不是炫耀，是清醒

### 趋势4：桌面悬浮类工具存在稳定需求

- Windows 11 / macOS Tahoe 持续改进桌面小组件体验，说明OS厂商也认可这个方向
- Catime（开源悬浮时钟）持续活跃，验证了"桌面常驻小工具"的使用习惯
- Chrome扩展类薪资时钟有5+个活跃产品，说明有人愿意为这个功能装扩展

## 🏢 三、行业格局

### 生产力赛道玩家分层

```
        企业协作 (Teams/Slack/Notion)
        ┌──────────────────────────────┐
        │      时间追踪 (Toggl/Clockify)│
        │  ┌────────────────────────┐   │
        │  │  个人小工具 (番茄钟等)  │   │
        │  │  ┌──────────────────┐  │   │
        │  │  │ 薪资时钟 ← 我们   │  │   │
        │  │  └──────────────────┘  │   │
        │  └────────────────────────┘   │
        └──────────────────────────────┘
```

> 我们在最底层、最细分的缝隙里。这个缝隙的好处是：**没有巨头会来做这个**。坏处是：需要自己教育市场。

### 关键竞争态势

| 维度 | 现状 |
|------|------|
| 巨头进入风险 | **极低**——太小众，太ToC，巨头看不上 |
| 创业公司竞争 | **低**——现有竞品都是个人项目/小团队，质量普遍不高 |
| 用户获取难度 | **中等**——没有现成的"时薪时钟"搜索需求，需要内容/口碑驱动 |
| 同质化风险 | **低**（短期）→ **中**（若成功）——智能三态检测有壁垒 |

## 💰 四、商业可行性

### 当前阶段：免费开源，积累用户

作为个人项目，MVP阶段不建议商业化。但中长期路径清晰：

| 阶段 | 策略 | 参考案例 |
|------|------|----------|
| MVP（现在） | 完全免费，开源，建立用户基础 | salary-timer |
| 增长期 | 免费+打赏/赞助 | Catime |
| 成熟期 | 免费核心功能 + 付费主题包 / Pro功能（云同步等） | 记工资（¥18 IAP） |

### 付费意愿参考

| 竞品 | 付费模式 | 用户反馈 |
|------|----------|----------|
| 记工资 | 免费+IAP ¥18 | 有付费用户，但广告惹众怒 |
| CashClock | 免费+广告 | 有付费去广告选项 |
| RealTimeCalc | 完全免费 | 未商业化 |

> **结论**：这个品类用户付费意愿存在但不高（¥18左右），靠广告会毁口碑，**建议长期走免费+主题/Pro增值路线**。

## 🎯 五、对项目的启示

### ✅ 该做的

| # | 行动 | 理由 |
|---|------|------|
| 1 | **押注"智能时间感知"** | 没有任何竞品做到，市场空白明确 |
| 2 | **坚持无广告** | 竞品最大槽点，我们的核心承诺 |
| 3 | **桌面浮窗优先** | 桌面小组件需求已被OS厂商和行为数据验证 |
| 4 | **打工人文化共鸣** | 不是冷冰冰的"计数器"，是"打工人的伙伴" |
| 5 | **先做Web版** | 零安装门槛，最大覆盖，最快验证 |

### ⚠️ 谨慎的

| # | 行动 | 理由 |
|---|------|------|
| 1 | **别急着做App** | 开发成本高，先用PWA验证移动端需求 |
| 2 | **别做社交/排行榜** | 隐私敏感，与"温和激励"调性冲突 |
| 3 | **别堆功能** | 竞品历史教训——记工资功能太多反而Bug多评分低 |

### ❌ 忽略的

| # | 行动 | 理由 |
|---|------|------|
| 1 | 企业版/SaaS化 | 与定位冲突，我们做ToC个人工具 |
| 2 | AI功能（现阶段） | 过度设计，MVP不需要 |
| 3 | 多语言国际化（现阶段） | 先做好中文市场 |

## 🔢 六、市场规模自下而上估算

针对"时薪桌面钟"的潜在用户池做一个地面估算：

```
中国城镇就业人口                  ≈ 4.7亿
其中"坐办公室的知识工作者"         ≈ 1.2亿（白领+自由职业）
对效率/财务工具有兴趣的比例        ≈ 15%（保守）
潜在目标用户                      ≈ 1800万

如果渗透率达到 0.5%（口碑传播）    ≈ 90万用户
```

> 这是天花板估算，不是目标。作为个人项目，**1000个忠实用户就是巨大成功**。但天花板够高，说明方向不窄。
