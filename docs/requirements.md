# 📋 时薪桌面钟 · 需求分析报告

> 生成日期：2026-06-24 | 版本：v1.1（新增日历检测）

---

## 🎯 产品一句话

**让打工人的每一秒都看得见价值——温和、数据化、有仪式感的时薪实时时钟。**

---

## 👤 目标用户

| 用户类型 | 特征 | 痛点 | 使用场景 |
|---------|------|------|---------|
| 🏢 办公室白领 | 朝九晚六，固定月薪，电脑前工作 | 摸鱼无感，不知道时间流逝的价值 | 桌面常驻浮窗，余光扫到数字跳动 |
| 🏠 自由职业/远程 | 工作时间灵活自定 | 缺乏上下班仪式感，需要自我激励 | 自定义工作时间段，满屏数据跳动 |
| 🎓 实习生/应届生 | 初入职场，对薪资敏感 | 月薪数字太抽象，感受不到"正在赚钱" | 看到每秒数字增长，增加价值感 |
| 💼 任何职场人 | 有固定月薪和工作时段 | 工作幸福感需要多维度提升 | 全场景：桌面挂件、手机查看、开始/结束提醒 |

---

## 📖 核心用户故事（User Stories）

| 编号 | 作为… | 我想要… | 以便… |
|------|-------|---------|-------|
| US-01 | 打工人 | 输入我的月薪、月工作天数、日工作时间段 | 系统能准确计算我的每秒薪资 |
| US-02 | 打工人 | 在桌面角落看到常驻浮窗 | 工作时余光扫到薪资增长，获得动力 |
| US-03 | 打工人 | 系统自动检测当前日期和时间状态 | 不用手动操作，打开就能用 |
| US-04 | 打工人 | 看到薪资像沙漏一样每秒累加 | 直观感受"时间就是金钱" |
| US-05 | 打工人 | 工作结束看到今日日薪总额 | 获得成就感和仪式感 |
| US-06 | 打工人 | 工作开始前看到倒计时 | 知道还有多久开始赚钱 |
| US-07 | 打工人 | 查看历史记录 | 了解昨天/本月累计赚了多少 |
| US-08 | 打工人 | 工作前看到 0.00 并自动等待 | 不用操心"现在计时对不对" |
| US-09 🆕 | 打工人 | 系统自动识别法定节假日和周末 | 休息日不会傻傻还在计算工资 |

## 🧩 功能分级

### 🔴 Must Have（MVP 核心）

| 功能 | 说明 | 对应故事 |
|------|------|----------|
| 💰 **薪资输入面板** | 月薪、月工作天数、日工作时间段（开始-结束时间） | US-01 |
| ⏱ **每秒实时累加** | 沙漏式数字跳动动画，每秒更新 | US-04 |
| 🕐 **智能三态检测** | 工作时→累加 / 工作前→0.00等待 / 工作后→日薪定格 | US-03, US-08 |
| 📊 **今日日薪展示** | 工作结束自动定格今日总额 | US-05 |
| ⏳ **工作前倒计时** | 显示距离工作时间开始还有多久 | US-06 |
| 📌 **桌面浮窗置顶** | Always on Top，角落常驻，不遮挡工作 | US-02 |
| 📜 **历史记录** | 存储每日薪资数据，可查看过往记录 | US-07 |
| 💾 **本地数据持久化** | 薪资设置+历史记录本地存储，刷新不丢失 | US-01, US-07 |
| 📅🆕 **法定节假日检测** | 内置年度节假日JSON + 调休安排，自动识别休息日 | US-09 |

### 🟡 Should Have（第一批迭代）

| 功能 | 说明 |
|------|------|
| 🔊 工作时间到提醒 | 工作时间开始时弹通知"开始赚钱了" |
| 🔔 工作结束提醒 | 结束时弹通知"今日收获 ¥XX.XX" |
| 📱 移动端适配 | 移动网页版响应式布局 |

### 🟢 Nice to Have（未来考虑）

| 功能 | 说明 |
|------|------|
| 🎨 **多主题/皮肤** | BCD混合调性——温和激励/冷淡数据/游戏趣味三种主题切换 |
| 🌙 暗色/亮色模式 | 跟随系统或手动切换 |
| 📈 月度统计图表 | 本月累计、日均、趋势折线图 |
| 🔢 多种薪资模式 | 月薪制 / 时薪制 / 年薪制 |

### ⚪ Won't Have（明确不做）

| 功能 | 说明 |
|------|------|
| 📤 数据导出/分享 | MVP阶段不做截图分享和CSV导出 |
| 👥 社交/排行榜 | 不做"好友排名""全国平均"等社交功能 |
| ☁️ 云端同步 | 纯本地，不登录，不上传数据 |

## 🎨 情绪调性

```
焦虑驱动 ❌          温和激励 ✅
冷淡数据 ✅          游戏趣味 ✅
```

> **BCD混合**：默认温和数据风，数字干净清晰，配合微妙的趣味动效（沙漏粒子感），用户可切换到不同主题。

---

## 🚧 关键假设与风险

| # | 假设 | 风险 | 验证方式 |
|---|------|------|----------|
| 1 | 用户愿意主动打开并设置参数 | 首次设置流程复杂可能导致放弃 | 简化输入为3步以内 |
| 2 | 用户会在工作时一直开着这个页面 | 页面被关闭后计时器中断 | 用 LocalStorage 记录状态，恢复时补偿计算 |
| 3 | "每秒跳数字"能持续吸引注意力 | 时间久了可能审美疲劳 | 通过动画微交互保持新鲜感 |
| 4 | 所有计算纯前端即可完成 | 无后端，数据仅存本地 | 使用 LocalStorage + Service Worker |
